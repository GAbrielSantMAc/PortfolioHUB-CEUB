# Regras de Contribuição para o PortfolioHUB

[cite_start]Este documento detalha o fluxo de trabalho e as regras para contribuição ao repositório PortfolioHUB-CEUB, garantindo o controle de versão[cite: 41].

## 1. Fluxo de Trabalho (Gitflow Simplificado)

Para garantir a segurança do código principal (`main`), todas as alterações devem seguir este fluxo:

1. **Criar Branch:** Não trabalhe diretamente na branch `main`. Crie uma branch de recurso (ex: `feature/nova-pagina`).
2. **Fazer Alterações:** Implemente o código, faça o Commit e o Push da sua nova branch.
3. **Abrir Pull Request (PR):** Abra um Pull Request (PR) para a branch `main`.
4. **Revisão:** O PR exige aprovação de um colaborador (Ação 7), garantindo que as políticas de segurança (Ação 6) sejam respeitadas.

## 2. Padrões de Commit

* **Clareza:** O Commit deve descrever **o que** foi feito.
* **Exemplos:** `feat: Adiciona integração com o LinkedIn`, `fix: Corrige erro na API`.
